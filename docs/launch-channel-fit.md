# Launch Channel Fit

MangoPods MCP should launch with two surfaces:

- Public demo mode - no login, no email, shows the workflow shape.
- Founder beta application - gated access for real MCP credentials, credits, partner scopes, and production usage.

This lets us collect qualified leads without weakening technical launches.

## Channel Rules

| Channel | Recommended URL | Why |
| --- | --- | --- |
| Hacker News Show HN | Public demo mode | Show HN needs something people can try. Do not submit a pure signup page. |
| Product Hunt | Main product page with demo and application | Product Hunt expects a product page, strong media, screenshots, and a clear CTA. |
| BetaList | Founder beta application page | Private beta and invitation-only access are a fit. |
| Indie Hackers | Founder story plus application | Best for feedback, use-case discussion, and build-in-public context. |
| Reddit | Native discussion post plus demo link | Lead with feedback and workflow questions, not a drive-by application link. |
| Uneed / DevHunt / Microlaunch | Demo page plus beta CTA | Treat as rehearsal launches with source-specific tracking. |
| MCP directories | Technical docs and tool catalogue | Only submit when tool schemas, auth, scopes, and connect instructions are clear. |

## Hacker News Posture

Use Show HN only if the public demo mode is visible and useful without login.

Good:

- Interactive source examples.
- Tool catalogue.
- Demo commands.
- Credit/approval model.
- Technical explanation in the first comment.

Bad:

- Signup-only landing page.
- "Join waitlist" as the only action.
- No concrete artefact to inspect.
- Marketing copy without implementation details.

Title options:

- Show HN: MangoPods MCP - media production tools for AI agents
- Show HN: I built an MCP surface for podcast, video, and campaign production
- Show HN: Give your agent a creative production backend through MCP

First comment structure:

1. What it is.
2. Why MCP.
3. What public demo mode shows.
4. What is still gated.
5. How credits, approvals, and publishing are controlled.
6. What feedback we want.

## Beta Application Scoring

Use the founder beta application to prioritise:

- High-reach users.
- Strong workflows.
- July testing capacity.
- Credit spend, resale, or margin potential.
- Case-study potential.

Do not optimise for the most signups. Optimise for the people most likely to make the product look inevitable.

## Suggested Launch CTA

Public demo CTA:

> Try demo mode.

Beta CTA:

> Apply for founder beta access.

Follow-up CTA:

> Tell us the workflow you would ship first.
