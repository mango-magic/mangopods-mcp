MangoPods apps request the smallest scope set that can complete the job. Read scopes let agents inspect existing projects and assets. Write scopes can create, render, publish, spend credits, or manage team operations.

Credit lifecycle.

Every meaningful job should start with an estimate. Agents can show the user expected spend, reserve a cap, execute the work, then settle or refund unused credits.

  1. Call credits.estimate with the tool family, input assets, quality tier, duration, language count, render target, and publishing destinations.
  2. Call credits.reserve with a maximum cap, user confirmation, and idempotency key.
  3. Pass credit_hold_id into the creative tool call.
  4. Watch the job until complete, failed, cancelled, or awaiting approval.
  5. Call credits.settle, or let MangoPods auto-settle once the job is complete.
{
  "tool": "credits.estimate",
  "arguments": {
    "operation": "video.generate_clips",
    "quality": "pro",
    "source_asset_ids": ["asset_raw_interview_01"],
    "outputs": [
      {"type": "vertical_clip", "count": 12, "duration_seconds": 45},
      {"type": "captions", "languages": ["en"]},
      {"type": "thumbnail", "count": 12}
    ]
  }
}

Assets and media plane.

MCP should not haul giant video files through the conversation. MangoPods uses an asset model: agents create upload sessions, import media from URLs, attach metadata, and receive signed URLs for download or preview.

Object Meaning Typical fields
project A creative workspace for an episode, client job, campaign, page, recording, or content sprint. project_id, name, owner, brand kit, status, due date.
asset A durable file or generated artifact such as raw media, rendered clips, transcripts, covers, captions, or exports. asset_id, type, mime, duration, dimensions, tags, signed URLs.
job An async operation that edits, renders, imports, analyzes, publishes, enriches, or generates something. job_id, state, progress, cost estimate, logs, result asset IDs.
recipe A reusable multi-step workflow that agents can run with user-approved inputs. recipe_id, version, steps, required scopes, credit policy.

Error model.

Errors are designed for agents. Every failure includes a stable code, a human-readable explanation, retry guidance, and a suggested next tool.

{
  "error": {
    "code": "credit_limit_exceeded",
    "message": "This render needs 184 credits, but the approved cap is 120.",
    "retryable": true,
    "suggested_next_tool": "credits.estimate",
    "safe_user_message": "The requested quality tier costs more than the approved budget. Try fewer clips, lower quality, or approve a higher cap."
  }
}

Security and reliability.

  • Use OAuth for user-facing products and API keys only for owned workspace automation.
  • Send an idempotency key for every spend, render, publish, booking, and import operation.
  • Prefer signed asset URLs with short lifetimes instead of sending raw media through chat history.
  • Use dry runs for destructive or expensive operations.
  • Request user confirmation for public publishing, paid boosts, high credit spend, and outreach campaigns.
  • Store webhook signing secrets outside the client and rotate them when a partner app is removed.

Build the wrapper once, monetise every recipe.

Most partner apps should not let agents call expensive tools naked. Wrap MangoPods with estimate, reserve, run, watch, settle, report, and notify. That gives users confidence and gives developers margin control.

Credit-safe TypeScript wrapper.

A partner app can expose one function that handles estimate, cap, reserve, run, and return job ID.

async function runMangoRecipe(client, input) {
  const estimate = await client.call("credits.estimate", {
    operation: "recipe.run",
    recipe: input.recipe,
    inputs: input.inputs
  });

  if (estimate.credits > input.maxCredits) {
    return {
      state: "needs_approval",
      message: `Estimated ${estimate.credits} credits. Approved cap is ${input.maxCredits}.`,
      estimate
    };
  }

  const hold = await client.call("credits.reserve", {
    estimate_id: estimate.estimate_id,
    approved_cap: input.maxCredits,
    idempotency_key: input.idempotencyKey
  });

  return client.call("recipe.run", {
    recipe: input.recipe,
    credit_hold_id: hold.credit_hold_id,
    inputs: input.inputs,
    approval_required: true
  });
}

Webhook-first fulfilment.

Use signed webhooks to keep your UI current even when the user closes the agent or the render takes an hour.

async function handleMangoWebhook(req, res) {
  const signature = req.headers["mangopods-signature"];
  const rawBody = await req.text();

  verifyWebhookSignature({
    rawBody,
    signature,
    secret: process.env.MANGOPODS_WEBHOOK_SECRET
  });

  const event = JSON.parse(rawBody);

  if (event.type === "job.succeeded") {
    await db.jobs.update(event.data.job_id, {
      state: "succeeded",
      outputAssetIds: event.data.output_asset_ids,
      settledCredits: event.data.settled_credits
    });
  }

  res.status(200).send("ok");
}

F A Q · T H E Q U E S T I O N S B U I L D E R S A S K

Answer the trust questions before they ask.

A vibe coder will try the page if the category is exciting. They will connect credits only if the money, files, publishing, and ownership model feel obvious.

Is this just a video editing API?

No. Video editing is one lane. The bigger idea is vibe broadcasting: source material becomes media, pages, posts, emails, booking flows, outreach, reports, and reusable assets.

Do giant files move through MCP?

No. MCP carries intent, IDs, constraints, approvals, and job state. Heavy media moves through signed upload sessions, imports, and asset URLs.

How do credits stay under control?

Every expensive job can estimate first, reserve with a cap, run asynchronously, settle actual usage, and refund unused credits.

Can my users bring their own MangoMagic credits?

Yes. Partner apps can use OAuth so users spend from their own workspace. Apps can also bundle credits, resell fixed jobs, or run agency margin models.

Can it publish without review?

Only when the workspace allows it. Public publishing, paid boosts, high credit spend, customer quotes, and private meeting reuse should support approval gates.

What should a developer build first?

Start with one narrow button: generate clips from a recording, build a page from a webinar, or create an episode launch pack. Prove value before exposing the whole toolbox.

Can agencies white-label this?

Yes. Agencies can create client workspaces, attach brand kits, run recipes, collect approvals, deliver assets, and report credit-backed margins.

What happens when a job fails?

The job returns a stable error code, logs, retry guidance, suggested next tool, and credit behaviour. The user sees a useful next step, not a dead end.

How does this fit Cursor, Claude, ChatGPT, or internal agents?

They connect to the same remote MCP server. The agent can read workspace context, estimate spend, import assets, run recipes, watch jobs, and return deliverables.

Why would users pay for this?

Because it turns expensive creative operations into outcomes they can request from any app: clips, pages, campaigns, launch packs, proof libraries, and reporting.

What makes it safe for private meetings?

Private meeting reuse can require consent, redaction, review queues, publish locks, retention rules, and audit trails before any public asset is created.

What is the best public demo?

Upload one messy recording. Ask an agent for a launch pack. Show the estimate, approval, running job, rendered clips, page, posts, and settled credits.
